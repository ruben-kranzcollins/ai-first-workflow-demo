# Project

## Goal

Demonstrate a lightweight AI-first workflow for our team.

## Scope

In scope:

- GitHub workflow demo
- branch protection
- pull request review
- shared Markdown files
- AI-use checklist
- safe handling of project context

Out of scope:

- real client data
- production code
- secrets
- deployment
- complex CI/CD
- enterprise governance

## Roles

| Role | Person | Responsibility |
|---|---|---|
| Project Owner | To be decided | Guards scope and acceptance |
| Repo Owner | To be decided | Manages repository settings and merge rules |
| AI/Tooling Owner | To be decided | Maintains AI context files |
| Reviewer | To be decided | Reviews pull requests |
| Security/Privacy Owner | To be decided | Checks data and safety rules |

## Data rules

Allowed:

- dummy data
- public information
- non-sensitive project documentation
- example code without secrets

Not allowed:

- API keys
- `.env` values
- PII
- client data
- production data
- contracts
- raw logs
- database dumps

## Definition of Done

A change is done when:

- it is linked to an issue or clear task;
- it is made in a branch;
- a pull request is opened;
- the PR-template is filled in;
- AI use is disclosed if applicable;
- at least one human reviewer approves;
- no sensitive data is included;
- relevant documentation is updated.
